Overview

What is Relatum and how does it work?

Relatum is an equational closure engine. Given an algebraic structure (operations + axioms) and a set of ground facts (equations between concrete terms), it derives all implied equalities by iteratively applying:

  • Reflexivity / Symmetry / Transitivity — handled implicitly by a union-find data structure
  • Congruence — if ai = bi for all i, then f(a1..an) = f(b1..bn)
  • Axiom instantiation — substitute axiom variables with every combination of known ground terms

The result is a set of equivalence classes: groups of terms that are provably equal under the given structure's axioms.

Quick Start

A minimal session to see the engine in action.

Step 1 — Load a Structure

Click Monoid in the Structure panel. This gives you:

  • Operations: mul/2, e/0
  • Axioms: associativity, left identity, right identity

Step 2 — Add a Seed Fact

Type a = a in the fact input and press Enter.

Why a = a? The engine only instantiates axiom variables with terms already in the universe. The fact a = a introduces the ground element a into the universe, making it available for substitution.

Step 3 — Compute Closure

Set rounds to 2 and click Compute Closure.

Result

-- Derived equations:
--   [identity]  mul(a, e) = a
--   [identity]  mul(e, a) = a
--   [identity]  mul(e, e) = e
--   [assoc]     mul(mul(a, a), a) = mul(a, mul(a, a))
--   ...

-- Equivalence class:
--   { a, mul(a, e), mul(e, a) }

Algebraic Structures

Operations, axioms, and built-in structures.

A structure consists of:

  • Operations — named functions with fixed arity (e.g. mul/2, inv/1, e/0)
  • Axioms (Equations) — universally quantified equalities (e.g. associativity)

Built-in Structures

NameOperationsAxiomsDescription
Semigroup mul/2 1 Associative binary operation
Monoid mul/2, e/0 3 Semigroup + identity element
Group mul/2, e/0, inv/1 5 Monoid + inverse operation
Ring add/2, zero/0, neg/1, mul/2, one/0 11 Abelian group (add) + monoid (mul) + distributivity
Empty none 0 Blank slate for custom structures

Axioms of a Group

-- Associativity
mul(mul(x, y), z) = mul(x, mul(y, z))

-- Left identity
mul(e, x) = x

-- Right identity
mul(x, e) = x

-- Left inverse
mul(inv(x), x) = e

-- Right inverse
mul(x, inv(x)) = e

Operations

Declaring and auto-registering operations.

Declaring Operations

Use the Add Op input with the format name/arity. Multiple entries can be comma-separated:

-- Declare a binary operation "star" and a constant "unit"
star/2, unit/0

-- Declare a unary operation
flip/1
ArityMeaningExample
0Constant (nullary)e/0 → written as e in terms
1Unary functioninv/1 → inv(x)
2Binary functionmul/2 → mul(x, y)
nn-ary functionf/3 → f(x, y, z)
Arity matters. A nullary operation e/0 is treated as a constant, not a variable. In terms, bare e is parsed as the constant e(), while bare x (not a declared operation) is a variable.

Auto-registration

When adding facts, any unknown identifier followed by parentheses is automatically registered as a new operation. The arity is inferred from the argument count:

-- Typing this fact on a Monoid automatically registers h/2:
h(a, b) = mul(a, b)

-- Registers g/1:
g(a) = a

Auto-registered operations appear with a purple border in the Structure panel, distinguishing them from built-in operations.

Arity conflicts. If you use the same name with different argument counts (e.g. f(a) then f(a,b)), the parser will raise an error.

Terms & Equations

Syntax, facts, and preset formulas.

Term Syntax

A term is one of:

FormMeaningExample
nameVariable (if not a nullary op)a, x, foo
nameConstant (if declared as arity 0)e, zero, one
name(t1, ..., tn)Operation applicationmul(a, inv(b))

Terms can be nested to arbitrary depth:

mul(mul(a, inv(b)), mul(c, e))

Adding Facts

A fact is an equation between two terms, written with =:

-- Simple seed: introduce a into the universe
a = a

-- Concrete identity instance
mul(a, e) = a

-- Claim commutativity for specific elements
mul(a, b) = mul(b, a)

-- Complex equation
mul(inv(a), mul(a, b)) = b

Each fact introduces all its subterms into the universe and asserts the two sides are equivalent. Facts can be deleted individually with the × button, or all at once with clear all.

Variables in facts. Variables in facts (like a, b) are treated as specific ground elements, not universally quantified. The fact mul(a, b) = mul(b, a) asserts that these particular a and b commute, not that multiplication is commutative in general. Universal quantification belongs in axioms (the structure's equations).

Preset Formulas (+)

The + button next to the input opens a dropdown of context-sensitive preset formulas. Each preset includes:

  • The formula text (fills the input on click)
  • A description explaining the algebraic meaning

Presets are grouped by category and adapt to the current structure's operations. For example, inverse-related presets only appear when inv is defined.

Closure Algorithm

How the closure engine computes equivalences step by step.

The closure computation proceeds in rounds:

  1. Seed — Collect all subterms from facts into the universe. Merge each fact's LHS and RHS in the union-find.
  2. Congruence — For every pair of f(a1..an) and f(b1..bn) in the universe where ai = bi for all i, merge them.
  3. Axiom instantiation — For each structure axiom, substitute its variables with every combination of ground terms from the universe. Merge the resulting LHS and RHS. New subterms are added to the universe.
  4. Repeat steps 2–3 for the configured number of rounds, or until a fixed point (no new merges in a round).
  5. Extract equivalence classes from the union-find (groups of terms with >1 member).
Exponential growth. Each round can dramatically increase the universe size because axiom instantiation generates |universe|^|vars| substitutions per axiom. Start with 1–2 rounds and increase carefully.

Safety & Limits

How the engine prevents combinatorial explosion.

Term Depth Cap (MAX_DEPTH = 8)

Terms nested deeper than 8 levels are never added to the universe. This prevents infinite growth from recursive axioms.

Static Expanding Axiom Detection

Before computing, the engine analyzes each axiom. An axiom is expanding if one side can be pattern-matched as a proper subterm of the other. For example:

Expanding axioms are flagged with a warning, and their substitution pool is restricted to shallow terms.

Dynamic Depth-Growth Detection

If the maximum term depth in the universe grows for 3 consecutive rounds and terms are being depth-capped, the engine halts early and reports a warning.

Other Limits

LimitValueEffect
MAX_UNIVERSE500Maximum distinct terms in the universe
MAX_DERIVED200Maximum derived equations before capping
MAX_SUBS_PER_AXIOM2000Maximum substitutions per axiom per round
MAX_DEPTH8Maximum term nesting depth
When limits are hit. The engine does not crash — it stops early and includes a warning in the result. You can still inspect partial results. If you see a limit warning, try reducing the number of seed facts or rounds.

Example: Double Inverse

Derive that inv(inv(a)) = a in a group.

-- Structure: Group (mul/2, e/0, inv/1)

-- Step 1: Seed
a = a

-- Step 2: Compute Closure (3 rounds)
-- The engine derives (among others):
--   mul(a, inv(a)) = e
--   mul(inv(a), a) = e
--   mul(inv(inv(a)), inv(a)) = e
--   inv(inv(a)) = a              ← double inverse!
--
-- Equivalence class includes:
--   { a, mul(a, e), mul(e, a), inv(inv(a)), ... }
How does this work? The engine instantiates the inverse axiom with x → inv(a), producing mul(inv(inv(a)), inv(a)) = e. Combined with mul(a, inv(a)) = e and congruence/transitivity, it concludes inv(inv(a)) = a.

Example: Zero Absorption

Derive mul(a, zero) = zero in a ring.

-- Structure: Ring
-- (add/2, zero/0, neg/1, mul/2, one/0)

-- Step 1: Seed
a = a

-- Step 2: Compute Closure (3 rounds)
-- The distributive law with y → zero, z → zero gives:
--   mul(a, add(zero, zero)) = add(mul(a, zero), mul(a, zero))
-- Since add(zero, zero) = zero (identity), we get:
--   mul(a, zero) = add(mul(a, zero), mul(a, zero))
-- Adding neg(mul(a, zero)) to both sides (via inverse axiom):
--   zero = mul(a, zero)

Example: Custom Structure

Explore a custom operation without any axioms.

-- Structure: Empty (click "Empty")

-- Step 1: Define operations
-- In "Add Op" input:
f/1, g/1

-- Step 2: Add facts
f(g(a)) = a        -- f is left-inverse of g
g(f(a)) = a        -- g is left-inverse of f
f(b) = g(b)          -- f and g agree on b

-- Step 3: Compute Closure (2 rounds)
-- Congruence derives:
--   f(f(b)) = f(g(b))    (since f(b) = g(b), apply f to both)
--   g(f(b)) = g(g(b))    (since f(b) = g(b), apply g to both)
-- Combined with the inverse facts:
--   f(f(b)) = b           (because f(g(b)) = b)
--   g(g(b)) = b           (because g(f(b)) = b)

Example: Recursive Detection

How the engine handles potentially infinite axioms.

-- Structure: Empty

-- Step 1: Define operation
f/1

-- Step 2: Add facts
f(a) = a
f(f(a)) = a

-- The engine does NOT infinitely nest f(f(f(...))).
-- Since f(a) = a and f(f(a)) = a, congruence gives:
--   f(a) = f(f(a)) = a
-- All collapse into one equivalence class:
--   { a, f(a), f(f(a)) }
With axioms (not just facts), expanding patterns ARE detected. If a structure axiom like f(x) = f(f(x)) is present, the engine recognizes it as recursive before computing and caps depth at 8. A warning is shown:

⚠ Axiom "wrap" is recursive (one side embeds in the other); depth capped at 8

Rust API

Using Relatum as a library in your Rust project.

Basic usage

use relatum::algebra::{builders, Equation, Term};
use relatum::algebra::closure::ClosureEngine;

// Build a monoid
let monoid = builders::monoid().unwrap();
let mul = monoid.find_operation_id("mul").unwrap();
let e_op = monoid.find_operation_id("e").unwrap();

// Create engine and add facts
let mut engine = ClosureEngine::new(monoid);
let a = Term::var("a");
engine.add_fact(Equation::new("seed", a.clone(), a.clone()));

// Compute closure (2 rounds)
let result = engine.compute_closure(2);

// Check result
let mul_a_e = Term::app(mul, vec![a.clone(), Term::constant(e_op)]);
assert!(result.equivalence_classes.iter().any(|c| {
    c.contains(&a) && c.contains(&mul_a_e)
}));

Inspecting results

// Inspect categories
for cat in &result.categories {
    println!("{}: {} equations", cat.rule, cat.equations.len());
    println!("  {}", cat.description);
}

// Check warnings
for w in &result.warnings {
    eprintln!("Warning: {}", w);
}

Equation with Category

use relatum::algebra::Equation;

// Equations can carry a category for grouped output
let eq = Equation::new("my_axiom", lhs, rhs)
    .with_category("my_category");

assert_eq!(eq.category(), "my_category");

Term Depth

use relatum::algebra::Term;
use relatum::algebra::operation::OperationId;

let f = OperationId(0);
let x = Term::var("x");

// x has depth 1
assert_eq!(x.depth(), 1);

// f(x) has depth 2
assert_eq!(Term::app(f, vec![x.clone()]).depth(), 2);

// f(f(x)) has depth 3
let ffx = Term::app(f, vec![Term::app(f, vec![x])]);
assert_eq!(ffx.depth(), 3);
